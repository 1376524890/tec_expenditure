# 效度检验与数据审计结论

## 1. 数据完整性

- 样本行数：4033。
- 企业数量：1666。
- 年份范围：[2019, 2020, 2021, 2022, 2023, 2024]。
- stock_code-year 重复记录：0。
- 必要字段齐全，可运行基准模型。
- 警告：样本未覆盖2015年前后，不能识别2015政策效果。
- 警告：样本未覆盖2018年前后，不能识别2018政策效果。
- 警告：省份匹配率低于80%，涉及省级财政变量的模型只能作为子样本分析。
- 警告：研发补助变量缺失率较高，补贴机制检验需使用缺失指示变量或子样本稳健性。

## 2. DID识别条件

- 当前政策冲击设定为 2021 年制造业研发费用加计扣除比例提高至100%，处理组为制造业企业，对照组为非制造业企业。
- 由于样本期为2019-2024，政策前仅2019、2020两个年份。事件研究只能提供非常有限的平行趋势证据，不能声称完成充分的动态趋势检验。
- 若 `event_m2` 即2019年相对2020年的处理组差异项显著，说明政策前趋势存在差异，应降低因果解释强度。
- 事件研究政策前项 event_m2：coef=0.0484, p=0.2561。

## 3. 稳健性分析规则

- 若基准模型、asinh替代因变量、剔除2024样本后的核心系数方向一致，说明结果较稳健。
- 若2022替代节点结果与2021节点差异较大，应解释为政策时点敏感，不应过度外推。
- 省级财政科技支出交互项只基于省份匹配子样本，不能代表全样本。

## 4. 税收优惠强度变量说明

- `rd_tax_deduction` 原始值中 99.8% 为负数，经 clip(lower=0) 和 log1p 转换后，仅约 0.2% 的观测有非零值（nunique=9）。
- 由于变量极度稀疏，`ln_rd_tax_deduction_w` 不进行 winsorize（winsorize 会使其完全退化为零常量）。
- M2 和 M3 中该变量的系数仅由 <1% 的非零观测驱动，解释时需极度谨慎，不能推广至全样本。
- 该变量不可直接解释为'企业享受了加计扣除'，仅反映原始数据中 rd_tax_deduction 为正的极少数情形。

## 5. 禁止性说明

- 不得把 `rd_expense_est` 写成企业真实研发费用。它是由加计扣除额反推的估算变量。
- 不得把 `rd_tax_deduction` 写成完整税务申报数据，除非原始数据说明确认为实际纳税调减额。
- 不得把累计专利数解释为当年新增创新产出。当前因变量是存量指标，短期政策响应可能被弱化。
