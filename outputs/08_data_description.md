# 数据说明

## 一、数据来源

### 核心数据
- **企业面板数据**: `data/firm_panel_final.xlsx`，中国A股上市公司年度数据，2019-2024年
- **省级面板数据**: `data/provincial_panel_full.csv`，31省年度数据，2011-2024年

### 省级R&D经费补充采集
- **来源**: 国家统计局、科学技术部、财政部《全国科技经费投入统计公报》（2018-2024年逐年发布）
- **采集年份**: 2017-2023年
- **采集方法**: 从公开统计公报中逐省提取"研究与试验发展（R&D）经费内部支出"（亿元）
- **完整度**: 198/217 = 91.2%（2017-2022年31省全覆盖，2023年覆盖12省）

## 四、关键数据问题

### 1. 省份匹配率低（48.08%）
- **原因**: 企业表中 `province_std` 字段缺失约52%的观测值
- **影响**: 所有涉及省级财政变量的模型只能使用约一半样本
- **解决**: 需从CSMAR公司基本信息表或Wind公司资料中补充企业注册省份

### 2. 税收优惠变量不可用
- **原因**: `rd_tax_deduction` 原始值99.8%为负数（均值为-3,268万元）
- 经clip(lower=0)和log1p转换后仅0.2%的观测有非零值
- winsorize会使其完全退化（1%和99%分位数均为0）
- **当前处理**: 不做winsorize，但该变量仅由8个观测驱动，p≈1.0
- **解决**: 需获取实际的企业加计扣除享受额（正值）

### 3. 研发补助缺失率高（40.8%）
- **影响**: 补贴机制检验需同时使用缺失指示变量和子样本
- **当前处理**: 缺失填0并加入missing指示变量

### 4. 缺乏真实研发费用
- **当前**: 仅有 `rd_expense_est`（由加计扣除额反推的估算值）
- **影响**: 机制检验不能区分"真实投入变化"和"会计处理差异"
- **解决**: 需从CSMAR研发创新库或年报补充真实研发费用

### 5. 因变量为存量指标
- **当前**: patent_stock 是累计专利存量
- **影响**: 对短期政策冲击的响应被稀释
- **解决**: 需补充 `invention_apply`（当年发明专利申请数）作为流量型因变量

### 6. 政策前窗口过短
- **当前**: 仅2019-2020两个政策前年份
- **影响**: 事件研究只能提供弱平行趋势证据
- **解决**: 需扩展面板至2017年或更早
